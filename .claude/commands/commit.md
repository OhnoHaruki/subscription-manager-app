# Gitmoji コミット作成

このスキルは、gitmoji と issue 番号を含むコミットを作成します。

## ルール

- コミットメッセージの先頭に適切な gitmoji を付ける
- issue 番号を末尾に付ける（例: `#1`）
- タイトル行の下に、行ったことを簡潔に箇条書きで記述する

## コミットメッセージ形式

```
<gitmoji> <タイトル> #<issue番号>

- 行ったこと1
- 行ったこと2
```

## 例

```
✨ サブスク一覧画面を追加 #1

- サブスク一覧を表示する ListScreen を実装
- ダミーデータで動作確認
```

## gitmoji 選定

https://gitmoji.dev/ を参照して適切な emoji を選ぶこと。よく使うものは以下の通り。

| emoji | 用途 |
| --- | --- |
| ✨ | 新機能の追加 |
| 🐛 | バグ修正 |
| ♻️ | リファクタリング |
| 📝 | ドキュメント |
| 🔧 | 設定ファイルの変更 |
| 🎨 | コード構造・フォーマットの改善 |
| ✅ | テストの追加・修正 |
| 🚀 | パフォーマンス改善 |
| 🔥 | コード・ファイルの削除 |
| 💄 | UI・スタイルの変更 |
| 🏗️ | アーキテクチャの変更 |

## 手順

1. ユーザーから issue 番号と変更内容を受け取る（引数がなければ `git diff --staged` で確認する）
2. 変更内容に適した gitmoji を選ぶ
3. 上記の形式でコミットメッセージを構成する
4. `git commit` を実行する

## 実行例

```bash
git commit -m "$(cat <<'EOF'
✨ サブスク一覧画面を追加 #1

- サブスク一覧を表示する ListScreen を実装
- ダミーデータで動作確認
EOF
)"
```

ユーザーが変更内容と issue 番号を伝えたら、適切な gitmoji を選んでコミットを実行してください。
issue 番号が不明な場合はユーザーに確認してください。
